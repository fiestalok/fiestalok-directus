# Send Mail Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend `/generate-pdf` to send the generated PDF by email to the client and an admin confirmation email to `contact@fiestalok.fr` via nodemailer + Hostinger SMTP.

**Architecture:** After uploading the PDF to Directus Files, `/generate-pdf` sends two emails (client + admin) using the PDF bytes already in memory. Email sending is best-effort — if it fails, the endpoint still returns `{ file_id }` with 200. No changes to the Directus flow (still one webhook).

**Tech Stack:** nodemailer, Hostinger SMTP (port 465/SSL), Jest + jest.mock for tests.

**Context:** The test file requires `./index` at module level. Jest hoists `jest.mock()` calls to before all requires, so we can mock nodemailer before `index.js` loads (which is when the transporter is created). - [ ] **Step 2: Add a helper function `mockHttpSequence` after the `app` require**

Insert this helper after `const app = require('./index');` and before `const VALID_KEY`:

```javascript
// Mocks sequential httpPost calls in index.js (Gotenberg, then Directus upload)
function mockHttpSequence(responses) {
  responses.forEach(({ statusCode, body }) => {
    http.request.mockImplementationOnce((opts, cb) => {
      const res = Object.assign(new EventEmitter(), {
        statusCode,
        statusMessage: 'OK',
      });
      const req = {
        write: jest.fn(),
        end: jest.fn(() => {
          const buf = Buffer.isBuffer(body) ? body : Buffer.from(JSON.stringify(body));
          res.emit('data', buf);
          res.emit('end');
        }),
        on: jest.fn(),
      };
      cb(res);
      return req;
    });
  });
}
```

    expect(mockSendMail).toHaveBeenCalledTimes(2);

    expect(clientCall[0].to).toBe('jean@example.com');
    expect(clientCall[0].subject).toContain('#42');
    expect(clientCall[0].attachments[0].filename).toBe('devis-42.pdf');

    expect(adminCall[0].to).toBe('contact@fiestalok.fr');
    expect(adminCall[0].subject).toContain('#42');
    expect(adminCall[0].attachments[0].filename).toBe('devis-42.pdf');

  test("un échec d'envoi email ne bloque pas la réponse — retourne file_id quand même", async () => {

      .post('/generate-pdf')

      .send({ reservation: VALID_RESERVATION });

    expect(res.status).toBe(200);
